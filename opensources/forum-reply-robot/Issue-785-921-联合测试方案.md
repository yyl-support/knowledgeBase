## 一、当前架构

```
外部调用方                     RAG API 层                   LightRAG
─────────                    ──────────                   ────────
POST /retrieve  ─→ OIDC 认证 → 限流 → 透传 ─→ POST /query
POST /paginated ─→ OIDC 认证 → 限流 → 透传 ─→ POST /documents/paginated
GET /status_counts → OIDC → 限流 → 透传 ─→ GET /documents/status_counts
GET /pipeline_status → OIDC → 限流 → 透传 ─→ GET /documents/pipeline_status
```

四个接口全部纯透传：请求体原样转发，响应原样返回。RAG API 层只负责认证 + 限流，不修改入参和响应。

```
/health + /health/detail    → 健康检查（无认证）
/metrics                    → Prometheus 指标（#785）
```

## 二、环境准备

```bash
BASE="https://<测试域名>"
TOKEN=""       # 通过 OIDC 获取
REFRESH=""     # 从 /auth/callback 返回
```

### 获取 token

浏览器访问 `$BASE/api/v1/rag/auth/authorize` → OneID 登录 → 回调返回 JSON → 复制 access_token / refresh_token。

## 五、Issue #921 测试 — 四个透传接口

### 5.1 retrieve 入参验证（POST → LightRAG `/query`）

核心目标：确认 RAG API 不修改调用方的入参，所有字段原样转发。

#### 5.1.1 完整入参

```bash
curl -s -X POST $BASE/api/v1/rag/retrieve \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "query": "BMC 固件升级流程",
    "mode": "mix",
    "only_need_context": true,
    "only_need_prompt": false,
    "response_type": "Multiple Paragraphs",
    "top_k": 5,
    "chunk_top_k": 10,
    "max_entity_tokens": 2000,
    "max_relation_tokens": 2000,
    "max_total_tokens": 8000,
    "conversation_history": [],
    "enable_rerank": true,
    "include_references": false,
    "stream": false
  }' | python3 -m json.tool
# 预期: 返回 JSON（LightRAG 原始响应），HTTP 状态码透传
```

#### 5.1.2 最简入参

```bash
curl -s -X POST $BASE/api/v1/rag/retrieve \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"query":"test"}' | python3 -c "import json,sys;d=json.load(sys.stdin);print('ok' if d else 'fail')"
# 预期: ok（JSON 解析成功），超时 60s
```

#### 5.1.3 入参字段不缺不漏（对照 LightRAG /query 文档）

| # | 字段 | 传入值 | 验证方式 |
|---|------|------|------|
| - | `query` (必填) | `"测试"` | 检索返回与测试相关的结果 |
| - | `mode` | `"mix"` | 响应正常，无报错 |
| - | `only_need_context` | `true` | 响应只有 context，无 LLM 回答 |
| - | `only_need_prompt` | `false` | 同上，false 时不只返 prompt |
| - | `top_k` | `1`, `5`, `10` | 多次调用，响应不同 |
| - | `chunk_top_k` | `5`, `20` | 同上 |
| - | `enable_rerank` | `true`, `false` | 同上 |
| - | `stream` | `false` | 不支持 stream，应正常返回 |
| - | `conversation_history` | `[]`, 含历史对话 | 带历史对话不报错 |

#### 5.1.4 边界值

```bash
# 空 query
curl -s -o /dev/null -w "%{http_code}" -X POST $BASE/api/v1/rag/retrieve \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" -d '{"query":""}'
# 预期: 透传给 LightRAG，由 LightRAG 决定拒绝或返回

# 超长 query（10KB）
curl -s -o /dev/null -w "%{http_code}" -X POST $BASE/api/v1/rag/retrieve \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d "{\"query\":\"$(python3 -c "print('A'*10240)")\"}"
# 预期: 不超时，60s 内返回（透传给 LightRAG 处理）

# top_k 极限值
curl -s -o /dev/null -w "%{http_code}" -X POST $BASE/api/v1/rag/retrieve \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" -d '{"query":"test","top_k":100}'
# 预期: 透传给 LightRAG
```

### 5.5 透传完整性总结

**RAG API 不对入参做任何修改**：不增加、不删除、不修改任何字段、不注入默认值。所有入参原样转发给 LightRAG。响应 HTTP 状态码和 JSON body 原样返回。

| # | 验证项 | 方法 | 通过标准 |
|---|--------|------|------|
| - | retrieve 入参透传 | 修改变量值后发起请求 | 响应随入参变化，RAG API 不报错 |
| - | paginated 入参透传 | 修改变量值后发起请求 | 响应随入参变化 |
| - | GET 接口无入参注入 | 发请求 | 无 query string 注入 |
| - | 响应状态码透传 | `curl -s -o /dev/null -w "%{http_code}"` | 与 LightRAG 直调一致 |

## 十四、一键验收脚本

```bash
#!/bin/bash
set -e
BASE="${1:?Usage: $0 <domain> <token>}"
TOKEN="${2:?Usage: $0 <domain> <token>}"
PASS=0; FAIL=0

check() {
  local code=$1 name=$2
  if [ "$code" = "200" ] || [ "$code" = "500" ]; then
    echo "✅ [$name] HTTP $code"; PASS=$((PASS+1))
  else
    echo "❌ [$name] HTTP $code"; FAIL=$((FAIL+1))
  fi
}

echo "=== 基础 ==="
check "$(curl -s -o /dev/null -w '%{http_code}' $BASE/health)" "health"
check "$(curl -s -o /dev/null -w '%{http_code}' $BASE/metrics)" "metrics"

echo "=== 认证 ==="
check "$(curl -s -o /dev/null -w '%{http_code}' -X POST $BASE/api/v1/rag/retrieve -H 'Content-Type: application/json' -d '{"query":"test"}')" "no-token"
check "$(curl -s -o /dev/null -w '%{http_code}' $BASE/api/v1/rag/documents/status_counts -H 'Authorization: Bearer fake')" "fake-token"

echo "=== 四个接口 ==="
check "$(curl -s -o /dev/null -w '%{http_code}' -X POST $BASE/api/v1/rag/retrieve -H "Authorization: Bearer $TOKEN" -d '{"query":"test"}')" "retrieve"
check "$(curl -s -o /dev/null -w '%{http_code}' $BASE/api/v1/rag/documents/status_counts -H "Authorization: Bearer $TOKEN")" "status_counts"
check "$(curl -s -o /dev/null -w '%{http_code}' $BASE/api/v1/rag/documents/pipeline_status -H "Authorization: Bearer $TOKEN")" "pipeline"
check "$(curl -s -o /dev/null -w '%{http_code}' -X POST $BASE/api/v1/rag/documents/paginated -H "Authorization: Bearer $TOKEN" -d '{"page":1}')" "paginated"

echo "=== 并发 ==="
RESULT=$(seq 20 | xargs -P20 -I{} curl -s -o /dev/null -w '%{http_code}' -X POST $BASE/api/v1/rag/retrieve -H "Authorization: Bearer $TOKEN" -d '{"query":"test"}')
F500=$(echo "$RESULT" | grep -c '500' || echo 0)
if [ "$F500" = "0" ]; then echo "✅ 并发无 500"; else echo "❌ 并发 $F500 个 500"; FAIL=$((FAIL+1)); fi

echo ""
echo "=== 结果: 通过 $PASS / 失败 $FAIL ==="
```
