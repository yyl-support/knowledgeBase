# 密钥与凭据管理

> 原则：**密钥、Token 等机密绝不明文存入本知识库或任何 git 仓库**。本笔记只登记「机密存在哪里、如何取用」，不含机密本身。

## 一、登记的凭据

| 条目 (svce) | 账户 (acct) | 用途 | 存放位置 | 登记日期 |
|------------|------------|------|---------|---------|
| `knowledgeBase-api-key` | `gorden` | API 密钥（`sk-` 形式） | macOS 钥匙串 (Keychain) | 2026-06-24 |

## 二、取用

需要明文时（会打印密钥，谨慎使用）：

```bash
security find-generic-password -a "$USER" -s "knowledgeBase-api-key" -w
```

在脚本 / 终端里注入为环境变量（推荐，不在命令行残留明文）：

```bash
export OPENAI_API_KEY=$(security find-generic-password -a "$USER" -s knowledgeBase-api-key -w)
```

只查看条目元数据（**不**显示密钥）：

```bash
security find-generic-password -a "$USER" -s "knowledgeBase-api-key"
```

## 三、更新 / 轮换

作废旧密钥并写入新值（`-U` 覆盖同名条目）：

```bash
security add-generic-password -a "$USER" -s "knowledgeBase-api-key" -U -w '<新密钥>'
```

> 建议：密钥一旦在聊天、日志、截图等场合暴露过，应立即去服务商后台作废并重新生成，再用上面命令覆盖。

## 四、删除

```bash
security delete-generic-password -a "$USER" -s "knowledgeBase-api-key"
```

## 五、为什么不放进知识库

- 本目录是 **git 仓库**，明文密钥一旦 commit/push 会永久留在历史中。
- Base64 等编码**不是加密**，任何人都能还原，只会制造虚假安全感。
- 钥匙串由操作系统加密托管，随用随取，且不落地为可被同步/提交的文件。
