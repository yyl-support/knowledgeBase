# 基础设施类 — 概览

> 分析时间：2026-06-10

## 基础设施在 opensourceways 组织中的定位

基础设施类仓库是整个组织的 **"底座"**，为上层所有业务服务提供：

- **服务注册与发现**：谁在哪里、跑什么版本、配置在哪
- **部署配置管理**：K8s YAML / Helm Chart 的统一存放与版本管理
- **配置安全管理**：Vault 敏感信息存储 + 预览环境注入
- **统一工具链**：社区服务 CLI、AI Agent 共享技能库
- **部署底座**：PostgreSQL 数据库、镜像仓库、K8s 集群管理

## 基础设施仓库清单

| 仓库 | 可见性 | 语言 | 核心职责 |
|------|--------|------|---------|
| `infrastructure` | 私有 | — | 服务部署档案总表（`service.md`）+ Vault 配置 |
| `infra-common` | 私有 | — | 部署 YAML 统一管理（kustomize/Helm） |
| `cora` | 公开 | Go | 统一社区服务命令行工具（OpenAPI 驱动） |
| `agent-skills` | 公开 | Python/Markdown | AI Agent 共享技能库（基础设施/上游/运营 三团队） |
| `integration-tests` | 公开 | Python/Shell | 跨仓端到端集成测试 |
| `secrets-manager` | fork | Go | Vault secrets → K8s Secrets 同步守护进程 |
| `karmada` | fork | Go | 多集群 Kubernetes 编排 |
| `ingress-nginx` | fork | Go | K8s Ingress 控制器 |
| `imagepullsecret-patcher` | fork | Go | K8s imagePullSecret 自动修补 |
| `trivy` | fork | Go | 容器镜像安全扫描 |

## 四个基础设施仓库的核心关系

```
infrastructure/service.md (服务档案)
        │
        ▼ 查 Vault Path
infrastructure/Vault (敏感配置)
        │
        ▼ 被 sync.sh 拉取
infra-common (部署 YAML + kustomize/helm)
        │
        ▼ 被 deploy.sh 使用
K8s 集群 (Pod/Service/Ingress)
        │
        ▼ 服务启动后
cora (CLI 工具访问各服务)
agent-skills (AI Agent 操作指南)
```
