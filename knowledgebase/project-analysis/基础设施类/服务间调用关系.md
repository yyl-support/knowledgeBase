## 二、逐条调用链详解

### 链①：infrastructure → Vault → 业务 Pod

```
backlog sync.sh
  → curl infrastructure/service.md (fetch)
  → 按匹配规则找到目标服务的 Vault Path
  → userpass 登录 Vault
  → curl $VAULT_PATH (read secrets)
  → Python 解析配置 / 改写为预览形态
  → kubectl create secret → 挂载到 Pod /vault/secrets/
```

**关键代码位置**（forum-reply-robot 的 sync.sh）：
- `:36-53`：读取 service.md，匹配 Vault Path
- `:62-69`：Vault userpass 登录
- `:70-73`：curl 读 Vault 配置
- `:80-117`：Python 解析 + 改写
- `:119-128`：烘 k8s Secret

### 链②：infra-common → K8s 集群

```
backlog deploy.sh
  → 从 infra-common 读取 Deployment/Service/Ingress YAML
  → 渲染 Pod 模板（按语言选基础镜像）
  → kubectl apply + rollout status
  → 写 deploy/<sub>.json 记录部署结果
```

### 链③：K8s → 业务服务 Pod → 运行依赖

```
Pod 启动后：
  git clone 业务源码 → pip/npm/go install → cp /vault/secrets/config → exec main.py
                                          ↑                        ↑
                                    来自 Vault (链①)      基础设施底座
                                                        PostgreSQL + SWR
```

### 链④：integration-tests → 业务服务

```
integration-tests/services/meeting-server/
  → run_all.sh 调用 meeting-server API
  → 验证服务功能正确性
```

### 链⑤：cora ↔ 各社区服务

```
cora CLI (开发者日常工具)
  → cora forum posts list      → Discourse API
  → cora gitcode issues list   → GitCode API
  → cora github prs list       → GitHub API
  → cora etherpad pads list    → Etherpad API
```

Cora 不参与自动化流水线，是**人工操作界面**。

### 链⑥：agent-skills → AI Agent

```
agent-skills 仓库
  → skills/infrastructure/  → 供基础设施团队 AI Agent 使用
  → skills/upstream/        → 供上游贡献团队 AI Agent 使用
  → skills/operation/       → 供运营团队 AI Agent 使用
  → skills/shared/          → 跨团队共享

接入方式：
  Claude Code: /plugin marketplace add opensourceways/agent-skills
  backlog 的 orchestrate.sh 中的 AI Agent 通过 README + AGENT.md 使用
```

```
                    infrastructure  infra-common  cora  agent-skills  integration-tests
```

## 四、数据流总结

```
1. service.md (infrastructure)  定义每个服务的完整元信息
2. sync.sh (backlog)            从 service.md 查 Vault Path → 拉取配置
3. deploy.sh (backlog)          从 infra-common 读部署 YAML → 部署
4. Vault                       存储 DB 密码、API Key、证书等敏感信息
5. K8s Secret (自动生成)        将 Vault 配置注入 Pod
6. cora                        开发者通过 CLI 与各服务交互
7. agent-skills                AI Agent 的操作知识库
8. integration-tests           验证服务功能正确性
```

整个基础设施体系的运转中心是 **backlog 的 `.ai-flow` 系统**，它读取 infrastructure + infra-common 的配置，驱动 Vault + K8s，支撑所有业务服务的开发与部署。
